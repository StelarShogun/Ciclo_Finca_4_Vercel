## Learned User Preferences

- Shared agent instructions also live in `CLAUDE.md`, `GEMINI.md`, `.github/copilot-instructions.md`, and `.cursor/rules/project-agent-rules.mdc`; keep them aligned when agent behavior changes.
- Write Git commit subjects/bodies and GitHub/GitLab PR titles and descriptions in English; Jira ticket IDs may appear alongside that text.
- When this stack runs in containers, run Laravel/PHP CLI (e.g. Artisan) inside the project Docker setup instead of assuming host PHP works the same.
- Always respond to the user in Spanish unless they explicitly request another language.
- Prefer phased Inertia/React storefront migrations: one route at a time, keep Blade until parity is validated, and avoid production-breaking big-bang rewrites.
- Prefer phased backend `app/` refactor (Actions, Services, Form Requests, `final readonly` DTOs) before more storefront frontend when the branch mixes legacy and new code; keep public routes and business rules unchanged; audit references before deleting classes.
- Defer focused frontend structural PRs (e.g. catalog page в†’ `features/client/catalog/pages/CatalogIndexPage.tsx`) instead of bundling them with backend refactors.
- Before merge to `Dev`, run `./scripts/ci-check-docker.sh` in Docker (Pint, MySQL PHPUnit, PHPStan, `npm run build`); browser smoke alone is insufficient for merge approval.

## Learned Workspace Facts

- The app uses Laravel with Vite and modular CSS bundles importing shared tokens from `variables-reset.css` (admin and client); there is no central Tailwind config driving the whole UI.
- CF4-132 green branding is modeled with `--brand-*` CSS variables; existing `--color-dark` and `--color-light` stay semantic neutrals in legacy rulesвЂ”do not remap them to brand greens without a deliberate token refactor.
- The storefront header wrapper class in Blade/CSS is `.cliente-header` (not `.client-header`).
- Client catalog work relies on a structural shell (`catalog-shell`, `catalog-hero`, `catalog-container`, category rail, filters, `catalog-main`); preserve established DOM ids and `data-*` hooks used by catalog JS, and keep presentation-only refactors from touching controllers, routes, queries, or APIs unless explicitly requested. UX: `footer.css` uses `--brand-*`; header catalog nav link and search apply on `clients.catalog` and `clients.product`; catalog search is a GET form in the client header on those routes; category trigger hidden from 1024px+ (rail + `catalog-sidebar-stack` sticky scroll); DM Sans on `.catalog-shell`; rail icons centered when collapsed; hero is an inset rounded card; filter submit uses `fa-sliders` not search icon; shared mobile/checkout chrome in `client-ux-improvements.css` (imported by `clients-page.css`).
- Overflow pitfalls on the storefront: category flyouts inside `overflow: auto`/`scroll` columns get clippedвЂ”move them out of the scroll subtree or change stacking; on catalog mobile (в‰¤1024px), `overflow-x: clip` on `html`/`body`/`.catalog-shell` hides horizontal scroll but exposes `--background-main` as a right-side gap when toolbar sort fields, long breadcrumbs, brand chips, spotlight Swiper, or mispositioned pagination `.sr-only` labels in `cf4-pagination-jump` exceed viewportвЂ”contain those children (position sr-only with inset) instead of relying on clip alone.
- Storefront checkout and performance: single-page cart (`cart.blade.php` or Inertia cart); step indicator via checkout progress partial; `mobile-back-nav` on product/cart/invoice; cart quantity typing debounced in `cart-actions.js` (~300ms); home loads `clients-home.css` not full `clients-page.css`; production long-cache on `/build/*` via `CacheStaticBuildAssets`.
- Admin category create/subcategory pages use `admin-shell` + shared SweetAlert (`data-cf4-confirm`); catalog import/export documented in `docs/CATALOG_IMPORT_EXPORT.md`. If category-trigger markup changes, refresh assertions in tests such as `CF4ClientCatalogCategoryMenuTest`.
- Dark mode: `html[data-theme="dark|light"]` with semantic tokens in client and admin `variables-reset.css`; preference persists in `localStorage` (`cf4-theme`) via `resources/ts/shared/theme-toggle.ts`вЂ”extend tokens/overrides in `variables-reset.css`, not color filters or global inversion. Client deep-green primary `#163832` / hover `#235347`; catalog surfaces use `--cf4-catalog-*`; admin overrides in `resources/css/admin/admin-dark-mode.css`. Every admin layout needs `admin.partials.cf4-theme-head` and `cf4-theme-scripts`; sidebar account menu (`admin.partials.sidebar-account-menu`) opens upward.
- Product card imagery: `ProductImageUrls::cardPicture()` serves `webp_768` then `webp_480` on mobile (в‰¤767px); reserve `webp_96` for small thumbnails (cart, favorites), not catalog/home cards.
- Client Inertia/React uses feature modules (`resources/ts/features/client/{home,catalog,product,cart,auth,...}`) plus `shared/`; `resources/ts/Pages` stays a thin Inertia entry (re-export pattern). Storefront is functionally complete on Inertia/ReactвЂ”remaining work is cleanup (legacy re-exports, shared Modal/Drawer backlog), not missing routes. Alert stack: Toast for simple feedback, InlineAlert for form/section errors, SweetAlert2 only for destructive confirms (cart clear, deletes, admin destructive)вЂ”not success toasts. Migrated routes include `/`, `/catalog`, `/legal/terminos`, product, cart, auth, profile, favorites, invoices, notifications; admin stays Blade except `/dashboard/inertia-pilot`. Inertia login/register keep standard reCAPTCHA (`g-recaptcha-response` when `config('recaptcha.site_key')` is set), not Enterprise.
- Backend `app/` layout: `Http/Controllers/{Admin,Client}/*`, `Actions/{Admin,Client}/*`, `Services/{Admin,Client,Security}/*`, `Data/*`; `Support` holds transitional helpers (`ProductImageUrls`, `ClientInertia/*`, ProductCatalogв†’`Services/Admin/ProductCatalog`). Fase 1 cart (`CartManager`, `app/Actions/Client/Cart/*`, `app/Data/Client/Cart/*`, `cartCount` as sum of line quantities); Fase 2 client auth/profile (Form Requests, Actions, `ClientAuthSessionState`). Dev-only Artisan: `dev:setup-database`, `dev:clean-demo-data` (local/testing only). No public route or business-rule changes during refactors.
- CI/CD and tests: `phpunit.xml` uses MySQL; `./scripts/test-mysql-docker.sh` в‰Ў `composer run test`; `composer run check` is faster with many skipsвЂ”not a pre-merge gate. `./scripts/ci-check-docker.sh` mirrors GitHub Actions. `Dev` branch CD deploys Render via Deploy Hook only when CI passes; PRs run CI but do not deploy. Dusk (`tests/Browser/`, local Chrome), Newman (`npm run test:api`, `postman/CF4-Storefront-API.postman_collection.json`), Pulse (`/pulse` admin, `PULSE_ENABLED=false` in PHPUnit). Email HTML previews: `php artisan mail:generate-previews`, not a CI test. See `docs/CI.md`.
